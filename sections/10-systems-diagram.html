Putting It All Together: End-to-End Data Flows

Inference Request Flow (North-South)

+
  1. User sends HTTPS request → enters via ISP A or B (BGP anycast)
  2. Hits DDoS scrubbing (Cloudflare) → clean traffic to Border Router
  3. Border router → Firewall HA pair (stateful inspection, ACLs)
  4. Firewall → Load Balancer (L7, TLS termination, health checks)
  5. LB selects healthy backend → routes to K8s Ingress Controller (NodePort/LoadBalancer service)
  6. K8s Ingress → Pod running inference server (e.g., Triton, vLLM)
  7. Pod's container has GPU access via nvidia-container-toolkit (VFIO or device plugin)
  8. Model loaded in GPU HBM3 memory → runs inference on H100 Tensor Cores
  9. Response travels back: Pod → Service → LB → Firewall → Border Router → ISP → User
  10. Total latency add: ~1-5ms network overhead on top of model inference time

Distributed Training Flow (East-West)

+
  1. User submits PyTorchJob via K8s API → Scheduler (Kueue/Volcano) gang-schedules 32 pods
  2. Each pod lands on a DGX H100 node with 8 GPUs allocated via Device Plugin
  3. Pods get secondary RDMA network interface via Multus + SR-IOV (ConnectX-7 VF)
  4. NCCL initializes: discovers NVLink topology (intra-node) + InfiniBand (inter-node)
  5. Training dataset loaded from Lustre/WekaFS via GPUDirect Storage → directly into GPU memory
  6. Forward pass runs on local 8 GPUs, connected via NVLink 4.0 (900 GB/s) through NVSwitch
  7. Gradient sync: NCCL AllReduce across 256 GPUs over InfiniBand NDR 400G
  8. Traffic path: GPU → ConnectX-7 → IB Leaf (QM9700) → IB Spine → Leaf → ConnectX-7 → GPU
  9. Adaptive routing on Quantum-2 switches distributes elephant flows across spine links
  10. Every N steps: checkpoint saved to Lustre parallel filesystem (striped across 8+ OSTs)
  11. Total GPU-to-GPU latency: ~1-3μs | Aggregate bisection bandwidth: 12.8 Tb/s

SRE Incident Response: GPU Node Down

+
  1. DCGM Exporter detects XID 48 (double-bit ECC error) on gpu-node-17, GPU 3
  2. Prometheus alert fires → AlertManager → PagerDuty page to on-call SRE
  3. SRE checks Grafana dashboard: GPU temp normal, but ECC uncorrectable error count rising
  4. SRE connects via BMC/Redfish (out-of-band): curl https://bmc-node17/redfish/v1/Systems/1
  5. Checks system event log: ipmitool -H bmc-node17 sel list confirms hardware fault
  6. Drain the node: kubectl drain gpu-node-17 --ignore-daemonsets
  7. K8s scheduler reschedules affected pods to other available nodes
  8. Running training jobs detect NCCL failure → framework restarts from last checkpoint
  9. SRE opens RMA ticket with NVIDIA for GPU replacement
  10. After hardware swap: run dcgmi diag -r 3 (full diagnostic) → uncordon node
